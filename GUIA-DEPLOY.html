1
Criar repositório no GitHub
Onde o código vai ficar hospedado · ~10 minutos
▼
⚠️ Se você não tem GitHub, crie uma conta grátis em github.com primeiro.

1. Instale o Git no seu computador

Acesse git-scm.com/downloads e baixe para Windows/Mac. Instale normalmente (next, next, finish).

2. Crie o repositório no GitHub

  1. Acesse github.com/new
  2. Nome: radar-invisivel
  3. Marque Private (código não ficará público)
  4. Clique em Create repository

3. Suba os arquivos do projeto

Abra o terminal (no Windows: clique com botão direito na pasta do projeto → "Git Bash Here" ou use o PowerShell) e rode:

cd radar-invisivel
git init
git add .
git commit -m "primeiro deploy"
git branch -M main
git remote add origin https://github.com/SEU_USUARIO/radar-invisivel.git
git push -u origin main
💡 Substitua SEU_USUARIO pelo seu usuário do GitHub.
Repositório criado no GitHub com os arquivos
Arquivos aparecem no GitHub (vejo a pasta api/, public/, etc.)
2
Configurar o Supabase (banco de dados)
Armazena usuários e oportunidades · ~15 minutos · Grátis
▼

1. Crie sua conta

Acesse supabase.com → Start for free → Entre com o GitHub.

2. Crie um novo projeto

  1. Clique em New Project
  2. Nome: radar-invisivel
  3. Crie uma senha forte e salve em algum lugar
  4. Região: South America (São Paulo)
  5. Clique em Create new project (aguarde ~2 minutos)

3. Crie as tabelas do banco

  1. No menu esquerdo, clique em SQL Editor
  2. Clique em New query
  3. Abra o arquivo database.sql que veio no projeto
  4. Copie todo o conteúdo e cole no editor
  5. Clique em Run (botão verde)
✅ Você deve ver "Success. No rows returned" — está correto!

4. Copie suas chaves de API

Vá em Settings → API (menu esquerdo) e copie:

URLhttps://XXXX.supabase.co — copie completo
anon / publicChave longa começando com eyJ... — é a ANON KEY
service_roleOutra chave eyJ... — é a SERVICE KEY (secreta!)
🔒 NUNCA coloque a SERVICE KEY no frontend. Ela só vai nas variáveis do Vercel.

5. Ative o Login com Google (opcional)

  1. Vá em Authentication → Providers
  2. Clique em Google
  3. Ative o toggle
  4. Você precisará configurar o Google OAuth Console (deixe para depois se quiser)
Projeto Supabase criado
SQL executado com sucesso (tabelas criadas)
Copiei as 3 chaves (URL, anon key, service key)
3
Configurar o Stripe (pagamentos)
Cobrança automática de R$9,99/mês · ~20 minutos
▼
⚠️ O Stripe não tem suporte direto a BRL nas contas novas. Use USD (U$ 1,99/mês ≈ R$9,99) ou considere o Mercado Pago para BRL nativo.

1. Crie sua conta no Stripe

  1. Acesse stripe.com → Start now
  2. Preencha seus dados (nome completo, CNPJ ou CPF, dados bancários)
  3. Complete a verificação de identidade

2. Crie o produto (plano Premium)

  1. No dashboard, vá em Products → Add product
  2. Nome: Radar Invisível Premium
  3. Descrição: Acesso ilimitado a oportunidades invisíveis
  4. Em Pricing: escolha Recurring
  5. Preço: 9.99 · Moeda: BRL
  6. Período: Monthly
  7. Clique em Save product
💡 Após salvar, você verá o Price ID no formato price_1ABC... — copie ele!

3. Copie as chaves de API do Stripe

Vá em Developers → API Keys:

Publishable keypk_live_... (pública, não usamos no backend)
Secret keysk_live_... — ESTA é a que vamos usar

4. Configure o Webhook

O webhook é o que avisa seu sistema quando alguém paga. Faça isso DEPOIS do deploy no Vercel (passo 5), mas anote onde fazer:

  1. Vá em Developers → Webhooks → Add endpoint
  2. URL: https://SEU-PROJETO.vercel.app/api/webhook
  3. Eventos a escutar: selecione "Select all events" ou escolha manualmente: checkout.session.completed invoice.payment_succeeded invoice.payment_failed customer.subscription.deleted customer.subscription.updated
  4. Clique em Add endpoint
  5. Copie o Signing secret (whsec_...)

5. Teste em modo sandbox primeiro

O Stripe tem um modo de teste. Use as chaves de teste (sk_test_...) durante o desenvolvimento e só mude para as chaves live quando tudo estiver funcionando.

Cartão de teste: 4242 4242 4242 4242 · Validade qualquer · CVV 123

Conta Stripe criada e verificada
Produto Premium criado com preço recorrente
Copiei: Secret Key e Price ID

5
Deploy no Vercel
Coloca o site no ar + configura variáveis · ~20 minutos · Grátis
▼

1. Crie sua conta no Vercel

  1. Acesse vercel.com
  2. Clique em Sign up → Continue with GitHub
  3. Autorize o Vercel a acessar seus repositórios

2. Importe o projeto

  1. No dashboard do Vercel, clique em Add New → Project
  2. Encontre o repositório radar-invisivel e clique em Import
  3. Em Root Directory: deixe em branco (raiz do projeto)
  4. Em Build & Output Settings: deixe tudo padrão
  5. NÃO clique em Deploy ainda — primeiro configure as variáveis abaixo

3. Configure as variáveis de ambiente

Esta é a parte mais importante. Clique em Environment Variables e adicione cada uma:

ANTHROPIC_API_KEYsk-ant-api03-... (da etapa 4)
SUPABASE_URLhttps://XXXX.supabase.co (da etapa 2)
SUPABASE_ANON_KEYeyJ... anon key (da etapa 2)
SUPABASE_SERVICE_KEYeyJ... service key (da etapa 2)
STRIPE_SECRET_KEYsk_live_... ou sk_test_... (da etapa 3)
STRIPE_PRICE_IDprice_1ABC... (da etapa 3)
STRIPE_WEBHOOK_SECRETwhsec_... (configurar depois)
NEXT_PUBLIC_URLhttps://radar-invisivel.vercel.app (URL do seu site)
⚠️ O STRIPE_WEBHOOK_SECRET você vai pegar depois do deploy. Coloque um valor temporário como placeholder e atualize depois.

4. Faça o Deploy

  1. Clique em Deploy
  2. Aguarde ~2 minutos
  3. Você verá uma URL como: https://radar-invisivel-abc123.vercel.app
🎉 Seu site está no ar! Agora vamos atualizar o frontend com as chaves públicas.

5. Atualize o arquivo index.html com suas chaves

Abra o arquivo public/index.html e encontre esta seção no início do JavaScript:

const CONFIG = {
  SUPABASE_URL:      'https://SEU-PROJETO.supabase.co',
  SUPABASE_ANON_KEY: 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.COLE-SUA-CHAVE-AQUI',
  API_URL: window.location.hostname === 'localhost'
    ? 'http://localhost:3000'
    : 'https://SEU-PROJETO.vercel.app',
};

Substitua os valores pelos seus reais, salve o arquivo e envie para o GitHub:

git add public/index.html
git commit -m "configura chaves"
git push

O Vercel vai fazer o redeploy automaticamente em ~1 minuto.

6. Configure o Webhook do Stripe agora

  1. Copie a URL do seu site: https://radar-invisivel.vercel.app
  2. Volte ao Stripe → Developers → Webhooks → Add endpoint
  3. URL: https://radar-invisivel.vercel.app/api/webhook
  4. Selecione os eventos (veja passo 3)
  5. Copie o Signing secret (whsec_...)
  6. No Vercel → Settings → Environment Variables: atualize o STRIPE_WEBHOOK_SECRET
  7. Vá em Deployments → clique nos 3 pontos → Redeploy

Colocar em produção (live)

  1. No Stripe, vá em Developers → API Keys e ative as Live keys
  2. Atualize as variáveis no Vercel: troque sk_test_... por sk_live_...
  3. Crie um novo webhook apontando para a URL de produção
  4. Faça redeploy no Vercel
⚠️ Só ative as live keys quando terminar todos os testes. Em live, pagamentos são reais!
